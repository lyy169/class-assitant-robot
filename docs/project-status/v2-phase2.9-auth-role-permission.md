## 8. Implemented Behavior

Database:

- Adds/updates only `users` and `teacher_classrooms`.
- Seeds `admin / admin123 / role=admin`.
- Seeds `teacher / teacher123 / role=teacher`.
- Binds teacher user to `classroom_101`.
- Passwords are stored as bcrypt hashes generated by `cloud_backend.security.hash_password`.

Auth API:

- `POST /api/auth/login` verifies username/password, sets HttpOnly `auth_token` cookie, and returns `redirect_to`.
- `GET /api/auth/me` resolves the current cookie or Bearer token.
- `POST /api/auth/logout` clears the `auth_token` cookie.

Page routing:

- `/` redirects by auth state and role.
- `/login` shows the unified login page when unauthenticated, otherwise redirects by role.
- Teacher pages require teacher/admin role.
- Admin pages require admin role.
- `/dashboard` requires login; admin can view all results, teacher is restricted to bound classrooms.

API permissions:

- `/api/admin/*` is admin-only.
- `/api/teacher/*` is available to teacher/admin.
- Teacher API results and classrooms are filtered by `teacher_classrooms`.
- `POST /api/interaction-results` remains unauthenticated in this phase.

Page identity:

- Teacher/admin/dashboard pages show `display_name`, `role`, and a logout button.
- 403 page is provided for role mismatch.

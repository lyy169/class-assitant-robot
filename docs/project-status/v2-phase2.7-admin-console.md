## 3. Added Admin APIs

### `GET /api/admin/overview`

Returns:

- `success`
- `admin`
- `metrics`
- `system_status`
- `status_distribution`
- `latest_results`
- `quick_links`

Metrics include:

- `teacher_count`
- `classroom_count`
- `result_count`
- `recent_result_count`
- `today_result_count`
- `raw_count`
- `reviewed_count`
- `archived_count`
- `avg_feedback_score`
- `avg_attention_score`
- `avg_response_score`

### `GET /api/admin/classrooms`

Parameters:

- `q`
- `teacher_id`
- `limit`
- `offset`

Returns:

- `success`
- `overview`
- `items`
- `total`

Each item includes classroom identity, teacher mapping, result count, average scores, status counts, latest result time, and `results_url`.

### `GET /api/admin/teachers`

Parameters:

- `q`
- `limit`
- `offset`

Returns:

- `success`
- `overview`
- `items`
- `total`

Each item includes teacher identity, username, classroom count, result count, average scores, latest result time, and `results_url`.

### `GET /api/admin/results`

Parameters:

- `classroom_id`
- `teacher_id`
- `status`
- `days`
- `limit`
- `offset`

Returns:

- `success`
- `filters`
- `overview`
- `items`
- `total`

Invalid `status` returns `400`. `limit` is capped at `100`. `days` uses the existing Phase 2.6 parser: `7`, `30`, or `all`.

## 4. Added Admin Pages

### `GET /admin`

Platform overview page with:

- Admin Console navigation
- platform status hero
- `Capture -> Local Analysis -> Cloud -> Teacher Feedback` pipeline
- core metric cards
- data ingestion status
- raw/reviewed/archived status distribution
- latest classroom analyses
- quick links

### `GET /admin/classrooms`

Classroom overview page with:

- Admin Console navigation
- classroom overview cards
- search and teacher filter
- classroom list
- recently active / performance ranking
- view results action to `/admin/results?classroom_id=...`

### `GET /admin/teachers`

Teacher overview page with:

- Admin Console navigation
- teacher overview cards
- search filter
- teacher list
- classroom-count ranking
- average-feedback ranking
- view results action to `/admin/results?teacher_id=...`

### `GET /admin/results`

All-platform results page with:

- Admin Console navigation
- result overview cards
- classroom / teacher / status / days / limit filters
- status distribution
- all classroom result list
- high-score / low-attention tips
- view analysis action to `/dashboard?result_id=...`

## 5. Data Aggregation

No database tables were added.

Aggregation reuses:

- `analysis_results`
- `sessions`
- `classrooms`
- `users`

Implementation notes:

- Admin view aggregates all visible platform results.
- `teacher_user_id` is used when present.
- Missing teacher mappings fallback to `Demo Teacher` / `demo`.
- `has_video` and `video_status` reuse Phase 2.5 display mapping.
- `system_status` is generated by lightweight queries from latest result data.
- Status distributions and rankings are computed server-side from existing rows.

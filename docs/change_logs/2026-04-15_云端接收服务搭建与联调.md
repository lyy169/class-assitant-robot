# 2026-04-15 云端接收服务搭建与联调

## 本次目标

为毕业设计“课堂交互分析系统”补齐云端部分的第一阶段能力，使本地高性能电脑能够将课堂交互分析结果通过 HTTP POST 推送到云服务器，并由云服务器完成接收、校验、落盘保存。

## 项目背景

系统整体采用三层架构：

1. 树莓派负责采集视频、抽取关键帧，并上传到本地高性能电脑。
2. 本地高性能电脑负责运行 YOLOv11 双类模型，对课堂交互行为进行检测，计算 20 秒窗口统计结果，并生成结构化 JSON。
3. 云服务器负责接收本地电脑推送的 JSON，为后续历史查询、仪表盘展示和聚合分析提供数据基础。

本次工作聚焦第 3 层，即云端接收服务。

## 联调过程中解决的问题

### 1. 依赖版本与镜像源不兼容

服务器使用的 Python 包镜像源不支持最初选择的部分最新版依赖，出现了以下问题：

- `uvicorn[standard]==0.35.0` 无法安装
- `pydantic==2.11.7` 无法安装

处理方式：

- 将 `uvicorn` 降为镜像源可用版本
- 将 `pydantic` 调整为镜像源可用版本
- 保持第一阶段目标以“先跑通链路”为优先，而非追求最新版本

### 2. Python 3.8 兼容性问题

服务器运行环境为 Python 3.8，而初始代码中使用了 Python 3.10+ 的类型注解写法，例如：

- `str | None`
- `dict[str, Any]`
- `list[Any]`

这导致 FastAPI + Pydantic 在应用启动阶段报错。

处理方式：

- 将关键模型和配置中的新语法替换为 Python 3.8 兼容写法
- 核心目标是确保服务能够稳定启动并响应请求

### 3. 端口占用问题

在重复启动服务时，出现了 `8010` 端口已被占用的问题。

这说明：

- 服务已经有实例在运行
- 新的实例因端口冲突无法再次绑定

该现象属于正常部署过程中的重复启动问题，不影响已经运行的服务实例。

# docs/changes

## 目录作用

`docs/changes/` 用于存放项目阶段性变更记录、联调总结、阶段报告和重要开发回顾，适用于树莓派端、本地高性能电脑端、云服务器端三端共用。

本目录中的文档主要回答以下问题：
- 本次改了什么
- 为什么要这样改
- 联调结果如何
- 当前遗留问题是什么
- 下一步计划是什么

## 使用规则

- 每次较大改动、跨端联调或阶段性里程碑完成后，必须新增或更新对应文档。
- 文档内容应尽量包含背景、目标、改动项、验证结果、风险与后续计划。
- 本目录记录“发生了什么”，不替代规格定义和接口设计说明。
- 若改动涉及跨端接口变化，必须同时更新 `docs/plans/` 中的相关接口约定文档。

## 文件命名规范建议

建议采用以下命名方式：

- `YYYY-MM-DD_主题.md`
- `YYYY-MM-DD_端_主题.md`

示例：

- `2026-04-15_云端接收服务搭建与联调.md`
- `2026-04-18_本地端_YOLO统计逻辑联调.md`
- `2026-04-20_跨端接口字段调整记录.md`

## 与 Constitution 的关联说明

本目录对应 Constitution 1.3 中的文档规范要求：

- 所有重要决策、阶段性改动和联调总结应统一沉淀在 `docs/changes/`
- 每次较大改动后必须补充文档记录
- 跨端协作时，变更记录必须与 `docs/specs/`、`docs/plans/` 保持一致

因此，`docs/changes/` 是项目开发留痕、回溯和答辩材料整理的重要基础目录。
